unreal-cli 커맨드 레퍼런스

Unreal Editor 5.3 · UnrealCliBridge Plugin · 총 38개 커맨드

전역 옵션

--json 응답을 JSON 포맷으로 출력 (--output json 단축)
--output default|json|compact 응답 포맷 지정
--project <path> 대상 Unreal 프로젝트 루트 디렉터리 지정
--force 파괴적 작업 또는 덮어쓰기 허용 (명령별 필요 여부 다름)
--wait compile / refresh 완료까지 대기 (해당 커맨드 전용)

에디터 제어 10개
status
언제든 가능
에디터 연결 상태, 현재 레벨, PIE 상태를 반환합니다.
play
PIE 시작
Play-In-Editor(PIE)를 시작합니다.
pause
실행 중인 PIE를 일시정지합니다.
stop
PIE를 종료합니다.
compile [--wait]
--wait 지원
Hot Reload 컴파일을 트리거합니다. --wait 시 완료까지 폴링합니다.
refresh [--wait]
--wait 지원
수정된 에셋을 에디터에서 재임포트합니다.
read-log [--limit N] [--type log|warning|error]
언제든 가능
Output Log 최근 항목을 읽어옵니다. 타입 필터링 가능.
screenshot [--viewport game|editor] [--path out.png] [--width N] [--height N]
게임 또는 에디터 뷰포트의 스크린샷을 캡처합니다.
execute-menu (--path "Menu/Item" | --list "Prefix")
메뉴 아이템을 실행하거나, 접두사로 등록된 메뉴 항목 목록을 조회합니다.
execute (--code <python> | --file <path.py>) [--args <json>] --force
--force 필수
Python Script Plugin을 통해 에디터 내에서 파이썬 코드를 실행합니다.
Python Script Plugin이 활성화되어 있어야 합니다. result 변수에 값을 할당하면 호출자에게 반환됩니다.

에셋 워크플로 7개
asset find [--name <term>] [--type <type>] [--folder /Game/...] [--limit N]
언제든 가능
이름·타입·폴더 조건으로 에셋을 검색합니다. --name 또는 --type 중 하나는 필수.
asset info --path /Game/...
언제든 가능
에셋의 메타데이터(타입, 패키지명, 디스크 경로 등)를 반환합니다.
asset move --from /Game/... --to /Game/... [--force]
에셋을 새 경로로 이동합니다. 대상이 이미 존재하면 --force 필요.
asset rename --path /Game/... --name <newName> [--force]
에셋 이름을 변경합니다. 동일한 이름이 존재하면 --force 필요.
asset delete --path /Game/... --force
파괴적 작업
에셋을 삭제합니다. 항상 --force가 필요합니다.
asset create --type <kind> --path /Game/... [--force]
새 에셋을 생성합니다. 대상이 이미 존재하면 --force 필요.
지원 타입: material, texture, staticmesh, blueprint, dataasset, curve, soundcue
asset mkdir --path /Game/...
Content 폴더를 생성합니다. 중간 경로도 함께 생성됩니다.
🗺️
레벨 워크플로 9개
level open --path /Game/... [--force]
레벨을 엽니다. 저장되지 않은 변경이 있으면 --force로 무시.
level inspect [--path /Game/...] [--with-values] [--max-depth N]
레벨의 액터 계층 구조를 반환합니다. --path 생략 시 현재 열린 레벨 대상.
level add-actor --class <ClassName> [--name <label>] [--location x,y,z] [--rotation p,y,r] [--scale x,y,z]
현재 레벨에 액터를 스폰합니다.
자주 쓰는 클래스: StaticMeshActor, PointLight, DirectionalLight, CameraActor, TriggerBox
level set-transform --actor <label> [--location x,y,z] [--rotation p,y,r] [--scale x,y,z]
액터의 Transform을 설정합니다. location·rotation·scale 중 하나 이상 필요.
level delete-actor --actor <label> --force
파괴적 작업
레벨에서 액터를 삭제합니다. 항상 --force 필요.
level list-components --actor <label>
언제든 가능
액터에 붙어있는 모든 컴포넌트 목록을 반환합니다.
level add-component --actor <label> --type <ComponentType> [--values <json>]
액터에 컴포넌트를 추가합니다. --values로 초기 프로퍼티 JSON 지정 가능.
level remove-component --actor <label> --type <ComponentType> [--index N] --force
파괴적 작업
액터에서 컴포넌트를 제거합니다. 항상 --force 필요.
level assign-material --actor <label> --material /Game/... [--slot N]
액터의 메시 컴포넌트에 머티리얼을 할당합니다.
🔵
블루프린트 워크플로 2개
blueprint inspect --path /Game/... [--with-values] [--max-depth N]
언제든 가능
Blueprint의 컴포넌트 계층 및 CDO 기본값을 조회합니다.
blueprint set-property --path /Game/... --property <name> --value <json>
Blueprint CDO(Class Default Object)의 프로퍼티 값을 설정하고 재컴파일합니다.

애니메이션 워크플로 5개
anim create-abp --skeleton /Game/... --path /Game/... [--force]
지정한 Skeleton으로 Animation Blueprint 에셋을 생성합니다. 대상이 존재하면 --force 필요.
anim assign-abp --bp /Game/... --anim-bp /Game/... [--component <name>]
Blueprint의 SkeletalMeshComponent에 Animation Blueprint를 연결하고 재컴파일합니다.
--component로 대상 컴포넌트 이름을 지정할 수 있습니다. 생략 시 첫 번째 SkeletalMesh가 대상.
anim list-states --path /Game/...
언제든 가능
Animation Blueprint의 상태머신 목록, 각 상태 이름, 정의된 변수 목록을 반환합니다.
anim add-variable --path /Game/... --name <varName> --type float|bool|int
Animation Blueprint에 변수를 추가하고 자동으로 재컴파일합니다.
지원 타입: float, bool, int
anim play-montage --actor <label> --montage /Game/... [--rate <float>]
PIE 필요
PIE 세션 중 지정한 액터에서 Animation Montage를 재생합니다.
활성 PIE 세션이 필요합니다. --rate 기본값은 1.0.

플러그인 관리 3개
plugin list
언제든 가능
현재 프로젝트의 모든 플러그인(활성·비활성 포함) 목록을 반환합니다.
plugin enable --name <PluginName>
플러그인을 활성화합니다. 에디터 재시작 후 적용됩니다.
plugin disable --name <PluginName> --force
파괴적 작업
플러그인을 비활성화합니다. 항상 --force 필요. 에디터 재시작 후 적용됩니다.

인스턴스 관리 & 진단 4개
instances list
알려진 Unreal 프로젝트 인스턴스와 현재 활성 선택 항목을 반환합니다.
instances use <projectHash|projectPath>
해시 또는 프로젝트 경로로 활성 대상 프로젝트를 고정합니다.
doctor
레지스트리, 프로젝트 감지, 라이브 도달 가능성 진단 정보를 출력합니다.
raw --json '{"command":"..."}' [--force]
로우 프로토콜 JSON 봉투를 직접 전송합니다. 저수준 디버깅 전용.
