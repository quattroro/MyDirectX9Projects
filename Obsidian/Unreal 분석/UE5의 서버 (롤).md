

| 롤                    | 설명                                                                                                                     |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| ROLE_None            | 액터가 롤을 갖지 않으므로 복제되지 않는다.                                                                                               |
| ROLE_Authority       | 현재 게임 인스턴스가 액터에 대한 완전한 권한을 찾는다. 즉 액터가 서버에 있으면 이 액터에 대한 변경 사항은 모든 클라이언트가 변수 리플리케이션을 통해 적용해야 하는 값으로 처리된다.                |
| ROLE_AutonomousProxy | 현재 게임 인스턴스가 액터에 대한 권한이 없지만, 플레이어 컨트롤러를 통해 제어된다. 즉, 액터 속도의 마지막 값을 사용하는 대신에 플레이어의 입력을 기반으로 좀 더 정확한 움직임 정보를 서버에 전송할 수 있다. |
| ROLE_SimulatedProxy  | 현재 게임 인스턴스가 액터에 대한 권한이 없고, 플레이어 컨트롤러를 통해 제어되지 않는다. 즉, 액터 속도의 마지막 값을 사용해 움직임을 시뮬레이션 한다는 것을 의미한다.                        |

## GetLocalRole()

현재 실행 중인 시스템(클라이언트 또는 서버)에서 객체가 어떤 역할(role)을 가지고 있는지 나타낸다.

## GetRemoteRole()

다른 시스템에서 이 객체가 어떤 역할을 가지는지 나타낸다.
객체가 서버와 클라이언트 간 복제될 때, 현재 시스템의 **반대쪽**에서 해당 객체의 역할을 나타냅니다.
